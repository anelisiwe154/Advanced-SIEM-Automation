# Kanban Board Explanation

It is a  visual project management tool that tracks tasks at different stages of the workflow . Tasks proceed between columns (such as To Do, In proceed, and Done) until they are finished.

## How the Board works
- Visualizes Workflow:  
  Columns represent the flow of work. Tasks move from Backlog -> To Do -> In Progress -> Review ->Testing -> Done. This makes progress visible and highlights bottlenecks.
  
- Limits work-in-Progress (WIP):  
  The set of limit is 3 tasks per column to prevent overload. This ensures the team focuses on finishing tasks before starting new ones, reducing bottlenecks and improving flow.
  
- Supports Agile Principles: 
  - Continuous Delivery: Tasks can be completed and delivered as soon as they reach Done, without waiting for a sprint to end.  
  - Adaptability: Columns can be customized to reflect team needs.  
  - Transparency: Everyone can see who is working on what, which supports collaboration and accountability.  

## Example
